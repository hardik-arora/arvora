🧠 Core Technology

What is a Radix Tree?

A Radix Tree (Compressed Trie) stores strings by merging chains of single-child nodes into compact edge labels — dramatically reducing memory and improving cache performance.

Standard Trie Uncompressed

root → m → u → m → [end]
              → b → a → i → [end]
              → b → l → e → [end]

⚠ 12 nodes for 3 words

Radix Tree Compressed

root → "mum" → [end]
           → "bai" → [end]
           → "ble" → [end]

✅ 4 nodes — 70% fewer!

Search complexity remains O(L) — same speed, far less memory.

🗄️ Dataset Pipeline

How the City Database Was Built

The 15,000-city dataset was prepared from the raw GeoNames geographical database for India.

1

Download & Extract

Fetched IN.zip from GeoNames containing all Indian geographical features in IN.txt.

2

Populated Place Filtering

Kept only Class P records (cities & towns). Removed lakes, monuments, and administrative regions.

3

Heuristic Sorting

Prioritized state and national capitals first, then sorted remaining cities by descending census population.

4

Clean & Deduplicate

Normalized to lowercase, collapsed spaces, removed punctuation, extracted top 15,000 unique city names.
